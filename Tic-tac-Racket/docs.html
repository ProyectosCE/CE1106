1 Descripción GeneralðŸ”—

Este programa implementa el juego de Tic Tac Toe (Tres en Raya) usando Racket y la librería ‘racket/gui‘. Los jugadores pueden jugar contra la computadora usando el teclado para seleccionar celdas y hacer sus movimientos. La computadora responde después de cada movimiento del jugador.

2 Archivos AdicionalesðŸ”—

El programa también depende de otros tres archivos que contienen funciones auxiliares y lógicas del juego:

- ""logic-utilities.rkt"": Utilidades para manipulación de listas y matrices. - ""game-status.rkt"": Funciones relacionadas con el estado del juego. - ""try-to-win.rkt"": Lógica del juego para que la computadora realice su movimiento.

3 Variables GlobalesðŸ”—

3.1 global-rowsðŸ”—

Número de filas del tablero de juego. El valor inicial es 3.

3.2 global-columnsðŸ”—

Número de columnas del tablero de juego. El valor inicial es 3.

3.3 blueðŸ”—

Color azul, utilizado para el carácter "'x'" (jugador humano).

3.4 redðŸ”—

Color rojo, utilizado para el carácter "'o'" (jugador computadora).

3.5 game-gridðŸ”—

Matriz que representa el estado del tablero de juego. Inicialmente está vacía y tiene dimensiones de global-rows por global-columns.

3.6 current-rowðŸ”—

La fila seleccionada actualmente en el tablero por el jugador.

3.7 current-columnðŸ”—

La columna seleccionada actualmente en el tablero por el jugador.

8 Clases y GUIðŸ”—

8.1 Clase canvas-box%ðŸ”—

procedure

(canvas-box% parent) → (is-a?/c canvas%)

  parent : any/c
Clase que representa una celda del tablero de juego. Cada instancia de esta clase es un lienzo donde se dibuja un carácter (ya sea un espacio, ’x’, o ’o’) y donde se capturan los eventos del teclado.

procedure

(on-char e) → void?

  e : event%
Captura eventos de teclado para navegar y seleccionar celdas en el tablero. Las teclas "w", "s", "a", y "d" se usan para moverse por las celdas, y "k" se usa para hacer un movimiento.

procedure

(on-paint) → void?

Dibuja el contenido del lienzo, resaltando si esta celda está actualmente seleccionada.

procedure

(set-color c) → void?

  c : color%
Establece el color de la celda (usado principalmente para resaltar la celda seleccionada).

procedure

(set-character char) → void?

  char : char?
Establece el carácter que se dibuja en el lienzo. El carácter puede ser un espacio vacío, "'x'", o "'o'".

9 Ventanas del JuegoðŸ”—

procedure

(TicTacToe) → void?

Inicializa el juego mostrando primero la ventana donde se eligen las dimensiones del tablero y luego la ventana principal del juego.

procedure

Reinicia el juego, limpiando el tablero y restaurando las variables globales. Resetea la matriz "game-grid" y refresca la interfaz gráfica.

procedure

(set-computer-move row-and-column) → void?

  row-and-column : (listof exact-integer?)
Realiza el movimiento de la computadora, actualizando la matriz "game-grid" y el carácter mostrado en la celda correspondiente.

procedure

(get-canvas-color character) → color%

  character : char?
Retorna el color asociado con el carácter proporcionado. Si el carácter es "'x'", retorna azul; para cualquier otro carácter, retorna rojo.
